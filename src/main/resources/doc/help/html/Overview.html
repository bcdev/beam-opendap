OPeNDAP client overview

The BEAM OPeNDAP client allows you to access an OPeNDAP server and download files using the data access protocol (DAP). The client offers the possibility to filter the displayed catalog entries by various characteristics.

Access to servers

You can enter the URL of an OPeNDAP server in the field at the top (please ensure the URL starts with "http://" and ends with "catalog.xml"). On either clicking "Return" or the refresh button at the left, the catalog content will be made accessible in tree structure. You can enter a new catalog URL at any time. VISAT will remember previously entered URLs.

Navigating through the catalog tree

The tree structure on the right displays the contents of the chosen server. You can open and close folders. Files which can be accessed via DAP are displayed as . When you select a node, information about it is displayed in the info boxes below the tree.
For files accessible via DAP, their DDS and DAS can be viewed (for more information about DDS and DAS please refer to http://www.opendap.org/pdf/dap_2_data_model.pdf ).
You can select one or more files in the catalog tree by clicking on them and pressing "Shift" or "Ctrl" at the same time.

Using the filters

With the filters, you can restrict the displayed files. A filter can be set after selecting the respective check box at the upper left. To apply the filter to the catalog, you must click the apply button. The filter will become inactive after deselecting the check box.
If you activate the check box, the previously defined filter will become active again. You can combine multiple filters to restrict your search for files.
The region filter and the variable filter both need information about the content of the catalog before they can be applied. You first need to browse through the folder structure to gather this information.

Using the dataset name filter

With this filter, you can search for whole file names or parts appearing in a file name. You can use wildcards:

  • * stands for any combination of signs
  • + stands for the last entered sign, appearing at least one time
  • ? stands for the last entered sign, appearing either once or not at all
For example, if you enter *2007* in the text field, the filter will filter out all files which do not have "2007" in their file name. If you enter *b+*, it will filter out all files which do not have a "b" in their file name.

Using the time range filter

This filter enables you to make a temporal restriction about the given files. The filtering is performed over the file name, so you need to know in which format the date is given in the names of the files over which you want to perform a filtering.
You can alter the date pattern, which says of which contents the date consists and in which order they appear in the file name. With the format pattern, you can specify where in the file name the date pattern is to be expected and how often it occurs.
You can set a start and end time for your filtering. Setting only a start time will filter out all files before the given time, setting only an end time will filter out all files after the set time, and setting both start and end time will filter out all files outside of the set temporal range.
Files of which no temporal information could be extracted will not be filtered out.

Using the region filter

The region filter allows you to define a region of interest. You can set this region by either changing the coordinate values in the text fields or dragging and resizing the rectangle on the world map. The filter will remove all files from the selection which are outside of the specified rectangle.
The filter is only applicable to files providing information about their geospatial extent in their metadata. Its checkbox can only be selected when such files are available in one of the folders of the catalog tree which has already been viewed by the user.
When a file's metadata has no information about its geospatial extent, it will not be affected by the filter.

Using the variable filter

This filter allows you to filter over the variables in the given files. Initially, this filter will have no information about these variables. When you browse through the folder structure, the folder will start scanning the opened folders for variables. After it has finished scanning, it will be available.
In the filter, you can check and uncheck the variables you are interested in. When applied, the filter will keep all files which contain at least one of the selected variables.

Downloading files

You can download the files selected in the tree by clicking the button titled "Download" in the lower right. When the client is run in VISAT, you can specify to open the products after download has completed. After you have set the target directory, the status bar will inform you about the progress of the download.
